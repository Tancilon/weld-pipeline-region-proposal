# Weld Seam Extraction Design

Date: 2026-04-11

## Overview

从焊缝OBJ网格中提取中心线，分段拟合为直线(line)和圆弧(arc)序列，输出结构化JSON描述文件。用于将CAD焊缝三维几何信息转换为紧凑的数学表示，便于后续焊接路径规划和位姿估计使用。

## 输入/输出

**输入**：
- 工件OBJ文件路径（用于提取model名称）
- 焊缝OBJ文件路径（Blender/SolidWorks导出的扫掠体网格）

**输出**：
- `{model}_weld_seams.json` — 焊缝描述JSON
- `{model}_weld_fit.png` — 中心线 vs 拟合结果对比图（可通过 `--no-viz` 跳过）

**JSON格式**：

```json
{
  "model": "工件1",
  "coord_system": "raw",
  "closed": false,
  "weld_seams": [
    {
      "type": "line",
      "points": [[x1, y1, z1], [x2, y2, z2]],
      "fitting_error_mm": 0.15
    },
    {
      "type": "arc",
      "points": [[x1, y1, z1], [x2, y2, z2], [x3, y3, z3]],
      "fitting_error_mm": 0.32
    }
  ]
}
```

- `coord_system: "raw"` — 保留原始CAD毫米坐标
- `closed` — 首尾端点距离 < 路径总长2%时为true
- 每段附带 `fitting_error_mm`，记录该段中心线点到拟合几何体的最大偏差

## 数据流

```
OBJ网格
  → OBJ解析（trimesh加载，选顶点数最多的object为主体）
  → PCA平面投影（拟合最佳平面，记录平面参数，投影到2D）
  → 中心线提取（mesh拓扑分析，截面环识别，环质心排序）
  → 曲率分段（离散曲率计算，阈值分类，合并短段）
  → 参数拟合（直线→2点，圆弧→3点，计算拟合误差）
  → 2D→3D反投影（利用平面参数映射回原始坐标）
  → 可视化 + JSON输出
```

## 模块设计

### 1. OBJ解析与主体对象选择

- 用trimesh加载OBJ，自动选择顶点数最多的object作为焊缝主体
- 过滤辅助几何体（如Cube等参考对象，通常顶点数极少）
- model名称从工件OBJ文件名提取，去掉扩展名

### 2. PCA平面投影

- 对焊缝主体所有顶点做PCA，得到3个主成分方向
- 第3主成分方向为平面法向量 `n`，顶点质心为平面原点 `o`
- 前两个主成分方向构成2D坐标轴 `u`, `v`
- 投影公式：`(dot(p-o, u), dot(p-o, v))`
- 保存平面参数 `(o, u, v, n)` 用于反投影：`p_3d = o + x*u + y*v`
- 质量检查：第3主成分方差占比超过5%时输出警告

### 3. 中心线提取

- 从三角面片中构建边邻接图
- 在2D投影空间中工作，识别扫掠体的截面环：
  - 选取起始顶点，通过BFS沿横向（垂直于路径方向）遍历同一截面环
  - 横向判断依据：相邻顶点连线方向与局部路径方向近似垂直
- 每个环取质心，按相邻关系排序成有序中心线点列
- 可选：轻度移动平均平滑，减少离散化锯齿
- 降级策略：若拓扑分析失败，退回到按角度排序、内外边界取平均的简化方法

### 4. 曲率分段

- 对有序中心线点列，用外接圆公式计算每个点的离散曲率 `κ = 1/R`
- 滑动窗口平滑曲率序列
- 自动阈值：基于曲率分布计算 `κ_threshold`
- 低于阈值 → 直线候选，高于阈值 → 圆弧候选
- 合并连续同类型点为一个段，过短的段（< 3个中心线点）并入相邻段
- 段边界点作为相邻段的共享端点，保证路径连续

### 5. 参数拟合

- 直线段：取首尾两点（3D坐标）
- 圆弧段：取首、中、尾三点（3D坐标），中点为该段中心线的中间点
- 计算每段中心线点到拟合几何体（直线/圆弧）的最大偏差作为fitting_error_mm

### 6. 闭合检测

- 计算首尾端点距离，小于路径总长2%时 `closed = true`

### 7. 2D→3D反投影

- 利用保存的平面参数将2D拟合关键点映射回原始3D坐标
- 反投影公式：`p_3d = o + x*u + y*v`

### 8. 可视化

matplotlib生成一张图，包含两个子图：

**左图 — 2D中心线 vs 拟合结果**：
- 灰色细线：原始中心线点
- 蓝色线段：拟合的直线段
- 红色弧线：拟合的圆弧段（密采样绘制平滑弧线）
- 绿色圆点：分段边界点
- 每段旁标注拟合误差值（mm）

**右图 — 3D视图**：
- 半透明灰色：原始焊缝网格
- 彩色线：反投影回3D的拟合路径

**终端输出摘要**：
```

```

## CLI接口

脚本路径：`scripts/extract_weld_seams.py`

```bash
python scripts/extract_weld_seams.py \
  --workpiece assets/工件1.obj \
  --weld assets/焊缝1.obj \
  --output assets/工件1_weld_seams.json
```

**参数**：
- `--workpiece`（必选）：工件OBJ路径，用于提取model名称
- `--weld`（必选）：焊缝OBJ路径
- `--output`（可选）：JSON输出路径，默认 `{workpiece同目录}/{model}_weld_seams.json`
- `--no-viz`（可选）：跳过可视化，仅输出JSON

## 技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 中心线提取方案 | mesh拓扑截面环分析 | 焊缝OBJ来自CAD扫掠操作，网格结构化，可直接利用 |
| 坐标系 | raw（原始CAD毫米） | 便于后续工程使用 |
| 分段精度控制 | 自动分段 + 可视化确认 | 先自动处理，输出对比图供用户判断 |
| 闭合处理 | 不强制闭合，JSON标识 | 尊重实际几何，`closed` 字段标识 |
| 批量处理 | 不需要，手动指定文件路径 | 当前需求简单，避免过度设计 |
| 主体对象选择 | 顶点数最多的object | 辅助几何体顶点数远少于主体，规则简单有效 |
